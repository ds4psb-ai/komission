# UI/UX Strategy (최신)

**작성**: 2026-01-07
**목표**: Komission의 Evidence Loop/캡슐 구조에 맞는 **현대적 UI/UX 원칙**과 화면 설계 기준을 고정

---

## 1) 리서치 요약 (최신 UI 흐름)
**2024~2025 트렌드 기준**
- **도구형 UI**: 긴 스크롤보다 **상태/맥락 기반 패널** + 탭 구조
- **Progressive Disclosure**: 필요한 순간에만 고급 정보 노출
- **Evidence-first**: 설명보다 **근거/지표가 먼저 보이는 구조**
- **시각효과 절제**: 글래스/3D/과한 애니메이션은 최소화
- **제품-운영 결합**: 데이터 테이블/로그가 사용 흐름에 자연스럽게 포함

**Apple 철학과 맞닿는 방향**
- **Clarity**: 텍스트/수치/CTA가 명확한 구조
- **Deference**: UI는 배경, 핵심 데이터가 전면
- **Depth**: 표면의 계층과 간격으로만 구조를 설명

## 2) 레거시 플랫폼 조사 요약 (공개 페이지 기반)
> 공개 접근 가능한 HTML 기준으로 분석했으며, SPA나 로그인 이후 영역은 제한됨.

### 2.1 리뷰플레이스 (reviewplace.co.kr)
- 상단에 **카테고리/타입 필터**가 과도하게 많음 (제품/지역/기자단/구매평 등)
- 캠페인 리스트가 **포인트/선정수/지원수 중심**으로 노출
- **창작자 중심 탐색형 UI**에 최적화되어 있고, 비즈니스 흐름은 분리됨

### 2.2 레뷰 (revu.net)
- SPA 기반, 정적 HTML에서 확인 가능한 키워드는 **체험단/회원가입**
- 설명 메타에 **블로그/인스타/유튜브/숏폼** 멀티 채널 강조
- **채널별 운영 중심**으로 보이며, Evidence/구조화 데이터는 약함

### 2.3 글로브 PR (globpr.co.kr)
- 서비스 메뉴에 **브랜드 스토리/서비스/인플루언서 마케팅** 중심 구조
- 캠페인 운영보다 **마케팅 서비스/컨설팅형 UI**에 가까움

**레거시 공통 문제**
- 크리에이터/브랜드 페이지가 분리되어 **맥락 이동이 많음**
- 캠페인 리스트가 과도해 **작업 단계(세션) 감각이 약함**
- “왜 이걸 해야 하는지”보다 “어디서 신청해야 하는지”만 강조됨

## 3) Komission 차별화 UI 방향
### 핵심 목표
- **세션 기반**: 사용자는 페이지가 아니라 **작업 진행 상태**를 본다
- **Evidence-first**: 숫자/패턴/근거가 먼저 보인다
- **캡슐 구조**: 복잡한 체인은 숨기고 **입출력만 노출**
- **Notebook Library는 백그라운드**: 결과는 Guide Node로만 노출
- **Role Switch**: Creator/Business 페이지 분리 대신 **스위치**로 전환

### Creator/Business 스위치
UI 상단에 토글로 역할 전환.
- Creator: Outlier 탐색 → Shoot → 제출/성과
- Business: Evidence → Decision → O2O/캠페인 운영

**장점**
- 동일 세션 컨텍스트 유지
- 이동 비용 최소화
- “페이지 분리”보다 운영 효율 상승

## 5) 화면 설계 원칙 (Apple-like)
### 5.1 레이아웃
- 헤더 + 세션 HUD + 본문(탭) 구조
- 카드형 계층 대신 **평면/얕은 그림자**
- 일관된 여백(8/16/24/32)

### 5.2 타이포그래피
- **KR**: Pretendard / SUIT (가독성 우선)
- **EN**: General Sans / Manrope (정제된 산세리프)
- 제목은 28–34, 본문은 15–16

### 5.3 컬러
- 베이스: **화이트 + 소프트 그레이**
- 포인트: **한 가지 컬러만 강하게 사용**
- “성과/리스크”는 색 대신 **아이콘 + 굵기 + 수치**

### 5.4 모션
- 페이지 진입 시 150–250ms 페이드/슬라이드
- 탭 전환에만 부드러운 트랜지션
- 노이즈/글래스/3D 틸트 효과 **금지**

## 6) 핵심 화면 스펙 (요약)
### Remix (Session)
- 상단: Session HUD (단계 + 다음 행동)
- 중앙: Evidence Summary + CTA
- 하단: Pattern 요약/리스크

### Shoot
- Hero CTA **최상단 고정**
- Variable Slot은 최소 입력만
- Quest/O2O는 칩 형태

### Business View
- Evidence Table → Decision Summary → O2O Matching
- 상세 설명은 접힘 상태

---

## 7) UI 구현 우선순위
1. **Role Switch + Simple/Pro 토글**
2. **Session HUD 고정**
3. **Evidence-first 카드 재배치**
4. **캡슐 노드 입출력만 노출**
5. **중복/과잉 효과 제거**
