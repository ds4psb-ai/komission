## 🚨 Top 3 Critical Issues

### Issue #1: 사용자가 "수익"을 못 본다
**문제**: 리믹스 상세 페이지에 "예상 수익 $X"가 없음  
**영향**: 사용자 동기 -80%

**증상**:
```
사용자: "이 영상을 촬영하면 얼마 버나요?"
앱: "..."
→ 포기
```

**해결**: [수익화 카드] 즉시 추가 필요
- UI: $10~30 예상액 표시 (카드 형태, 초록/빨강 gradient)
- 데이터: Neo4j에서 유사 fork 50개 평균값
- 배치: AI 가이드 위 (사용자의 "왜"를 먼저 해결)

---

### Issue #2: 촬영 후 "축하"가 없다
**문제**: 촬영 완료 후 축하 모달, 수익 예상액 표시 없음  
**영향**: 마이페이지 진입율 -85%, 재방문율 -70%

**증상**:
```
사용자가 촬영 완료하면 어디로?
→ 막힘 → 나감
```

**해결**: [축하 모달] + [수익 확인] CTA 필요
- Confetti 애니메이션 + 큰 텍스트
- 수익 예상액 표시 ($10~30)
- K-Points 즉시 지급 (기본 350P + 퀘스트 500P)
- [내 수익 확인] CTA 강조 (기본값 선택)

---

### Issue #3: O2O 매칭이 불명확하다
**문제**: 퀘스트 추천이 리믹스와 자동으로 연결되지 않음  
**영향**: 수익화 경로 -60%

**증상**:
```
사용자: "어디서 돈을 버지?"
앱: "마켓에서 퀘스트를 찾아봐"
→ 복잡함 → 포기
```

**해결**: 자동 매칭 API 필요
```
POST /api/v1/remix/{nodeId}/matching
{
  category: "fashion",
  music_genre: "k-pop",
  platform: "tiktok"
}

Response:
{
  recommendedQuests: [
    { brand: "ZARA", incentive: 500 },
    { brand: "MAC", incentive: 300 }
  ]
}
```

## 🎓 핵심 교훈

### "왜 현재 상태에서 사용자가 안 돌아올까?"

1. **"수익"이 안 보임**
   - 사용자는 "돈이 나오는지" 먼저 확인함
   - 수익화 카드 없이는 동기가 -80%

2. **"행동" 후 "반응"이 없음**
   - 촬영 완료 후 축하가 없음
   - 심리학: Peak-end rule (마지막 순간이 전체를 결정)

3. **"루프"가 자명하지 않음**
   - "다음은 뭐 하는가?"가 불명확
   - 자연스러운 흐름이 없음

### 해결책: 3가지 신호

```
사용자 심리: 
"이게 나한테 도움이 되나?" 
     → "정말 돈이 나오나?" 
     → "계속 할 이유가 있나?"
        ↓              ↓              ↓
   [동기 신호]   [수익 신호]   [행동 신호]
        ↓              ↓              ↓
  [리믹스 상세]  [축하 모달]   [다음 리믹스 CTA]
 (수익화 카드) (수익 예상액)  (게이미피케이션)
```
